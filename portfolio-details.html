Project Summary

This is one type of search engine but related to the chemical industry. Only chemical related terms you can search on it i.e butane, methane hydrocracking etc. This search engine is created for the use of some specific persons only who are working on that company. And mostly those persons are Scientist or Researcher in that chemical industry.

Requirements

Actually, those Scientist, Researcher or some other persons who completed the Ph.D in chemical firms, they all have created many documents. And while working they want to find some details of any chemical entity then they have to open those all related documents and read it from it. What if there are many related/required documents i.e. 1000 documents then basically no one will open those 1000 documents and read from it. So, the requirement is to extract the information like Abstract, author name, date of publication, images, region etc. of the document and store it into the database. So, when user will search any chemical term then we have to find that term in different document and based on it show all the information in UI like abstract, date, author name, document link, region, some special chart that is derived from the searched result like Document type and count, author and count, region and count, year and count etc. Also users can filter the result by author, date, region etc.

Technology, Libraries/Frameworks:

Python 3 | Django | Django REST Framework | Apache Solr | HTML | CSS | Javascript
